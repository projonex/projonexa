## About Projonexa

**Projonexa** is a technology-driven innovation platform that helps students, developers, innovators, startups, researchers, and businesses transform ideas into real-world solutions.

Many students have innovative project ideas but lack technical expertise, development resources, documentation support, research guidance, time, or mentorship. **Projonexa bridges this gap** with complete end-to-end project development.

| Metric | Impact |
|--------|--------|
| **100+** | Projects Delivered |
| **150+** | Freelancer Network |
| **10+** | Technology Domains |
| **500+** | Students Assisted |

**Who we serve:** Students · Colleges · Startups · Researchers · Freelancers · Businesses · Industry professionals

## Getting Started

### Prerequisites

- **Node.js** 20+ ([`.nvmrc`](.nvmrc))
- **npm** 9+

### Installation

```bash
# Clone the repository
git clone https://github.com/nikobuddy/projonexa.git
cd projonexa

# Install dependencies
npm install

# Start development server → http://localhost:3000
npm run dev
```

### Build & Preview

```bash
# Production build (output: .next/)
npm run build

# Run production server locally
npm start
```

### Environment Variables

Copy the example file for local configuration:

```bash
cp .env.example .env.local
```

Use `NEXT_PUBLIC_*` only when a value must be exposed to the browser. Never commit `.env` or `.env.local`.

### Deployment (Vercel — recommended)

The site is configured for **[Vercel](https://vercel.com)** via [`vercel.json`](vercel.json):

- **Build:** `npm run build` → `.next/` (sitemap, OG images, SEO validation, static/SSG pages)
- **SEO:** Server-rendered metadata + JSON-LD per route; `/sitemap.xml` and `/robots.txt` from App Router
- **Redirects:** `/final-year-projects` → `/college-projects`, `/research` → `/services`

**Quick setup**

1. Import the GitHub repo in Vercel  
2. If the repo root is the parent folder, set **Root Directory** to `projonexa-main-website`  
3. Deploy (Vercel reads `vercel.json` automatically)  
4. Add custom domain `projonexa.com` in project settings  

Full guide: [`docs/VERCEL_DEPLOYMENT.md`](docs/VERCEL_DEPLOYMENT.md)

Optional Netlify config: [`docs/deployment/netlify.toml.example`](docs/deployment/netlify.toml.example)

See [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) for architecture and SEO notes.

## Project Structure

```
projonexa-main-website/
├── public/                 # Static assets, OG images, sitemap.xml, llms.txt
├── scripts/                # Build-time SEO & asset generators
├── src/
│   ├── app/                # Next.js routes, layouts, metadata
│   ├── components/         # Layout, sections, forms, UI, SEO helpers
│   ├── data/               # Content & PAGE_SEO
│   ├── lib/seo/            # Metadata, JSON-LD, sitemap helpers
│   ├── theme/              # Design tokens & Tailwind preset
│   └── views/              # Page UI (wired by app routes)
├── docs/                   # Architecture, brand, SEO, deployment
└── .github/                # CI, templates, Dependabot
```

Full tree → [`docs/PROJECT_STRUCTURE.md`](docs/PROJECT_STRUCTURE.md) · Architecture → [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md)
