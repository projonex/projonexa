# Scripts

Node scripts run during **build** or **maintenance**. All use ES modules (`"type": "module"` in `package.json`).

| Script | npm command | Purpose |
|--------|-------------|---------|
| `generate-sitemap.mjs` | `npm run sitemap:generate` | Writes `public/sitemap.xml` from `src/lib/seo/sitemap-urls.ts` |
| `validate-seo.mjs` | `npm run seo:validate` | Validates `PAGE_SEO` against `src/lib/seo/seo-rules.ts` |
| `generate-og-images.mjs` | `npm run og:generate` | Renders `public/og/*.png` social preview images |
| `generate-brand-icons.mjs` | `npm run brand:icons` | Regenerates favicon / PWA icons from logo |
| `export-ctr-variants.mjs` | `npm run seo:ctr:export` | Exports title/description variants for CTR tests |
| `create-github-issues.mjs` | â€” | One-off GitHub issue bootstrap (optional) |

**Build pipeline** (`npm run build`):

1. `sitemap:generate`
2. `seo:validate`
3. `og:generate`
4. `next build`

Runtime SEO routes (`/sitemap.xml`, `/robots.txt`) are generated by Next.js from `src/app/sitemap.ts` and `src/app/robots.ts`. The static `public/sitemap.xml` is kept for tooling and hosts that read the file directly.
